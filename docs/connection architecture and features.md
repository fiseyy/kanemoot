# Архитектура подключения, режимы и плюшки

## 1. Режимы работы

**Режим работы сервера:**
- `Главный (General)` — основной сервер `kanemoot.ru`. Доступ к глобальной сети.  
- `Приватный (Private)` — локальный или корпоративный сервер. Полностью изолированный, используется для приватного общения.

---

## 2. Тип подключения

| Тип подключения                  | Режимы применения       | Плюшки | Примечания |
|---------------------------------|------------------------|--------|------------|
| `Прямое`                         | Главный                | ❌      | Подключение напрямую к `kanemoot.ru` |
| `Через крупный прокси (CDN)`     | Главный                | ❌      | Fallback, если прямое подключение недоступно |
| `Публичный self-hosted VPS`      | Главный                | ✅      | Пользователь открыл VPS для всех. После проверки доступности выдаются плюшки (SP, значок, аним. рамка) |
| `Приватный self-hosted VPS`      | Приватный / Главный    | ❌      | Подключение только для себя или своей группы. Полная приватность |

> **Примечание:** проверка публичного VPS включает пинг к основному серверу, открытость порта, доступность со стороны других клиентов.

---

## 3. Плюшки и SP (Support Points)

### 3.1 За публичный VPS
- Значок `Поддержал проект`.
- Анимированная рамка вокруг аватара.
- SP начисляются ежемесячно.
- Доступ к экспериментальным фичам (ранний доступ к новым UI, режимам чата и функциям).

### 3.2 За подписку
- Поддержка проекта и основного VPS.
- SP начисляются ежемесячно.
- Доступ к кастомизации профиля:
  - Рамки аватара
  - Мини-фоны
  - Полноразмерные фоны профиля
- Доступ к экспериментальным фичам.
- Может суммироваться с плюшками от публичного VPS.

### 3.3 Приватные VPS / прямое подключение / прокси
- Плюшек нет.
- Используется для приватности и стабильности соединения.

## 4. Логика начисления SP и плюшек
```

                 +----------------+
                 | Главный сервер |
                 +--------+-------+
                          |
                          v
                    +-----+-----------+
                    | Тип подключения |
+-------------------+-----------------+-+
|                   |                   |
v                   v                   v
Прямое          CDN прокси        Self-hosted VPS
                                   /         \
                                Private     Public
                                   |          |
                                   ❌         ✅
                                       SP + Значки
                                       Аним. рамка
                                   Эксперименталка
```

## 5. Итоговая таблица

| Действие / Подключение       | Плюшки | Тип плюшек |
|-------------------------------|--------|------------|
| Прямое к kanemoot.ru            | ❌      | —          |
| Через CDN прокси                | ❌      | —          |

| Подписка                        | ✅      | SP, кастомизация профиля, эксперименталка |

## 6. Ключевая концепция

- `Публичные VPS` — расширяют сеть канемута, поддерживают стабильность для всех пользователей. Плюшки мотивируют пользователей открывать свои VPS.  
- `Приватные VPS` — полностью контролируемое, безопасное и приватное подключение. Пользователь сам решает, кто подключается.  
- `SP и плюшки` — механизм мотивации и вознаграждения. Комбинируются между подпиской и публичными VPS.  
- `Fallback-схема` — обеспечивает стабильность соединения через:  
  `Прямое подключение → CDN → публичные VPS`.
